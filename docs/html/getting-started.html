Getting Started

This guide walks you through system requirements, installation, your first launch, and the key concepts you need to understand MultiTerminal.

System Requirements

Requirement Details
Operating System Windows 10 or Windows 11 (x64)
Runtime .NET 8 (bundled with the application)
WebView2 Runtime Microsoft Edge WebView2 Runtime (usually pre-installed on Windows 10/11)
Disk Space ~400 MB
Claude Code Required for agent features. Install via npm install -g @anthropic-ai/claude-code
Node.js Required for the MCP server (v18+ recommended)

If WebView2 is not installed, download it from https://developer.microsoft.com/en-us/microsoft-edge/webview2/.

Installation

  1. Download the latest MultiTerminal release archive.
  2. Extract the archive to a folder of your choice (e.g., C:\MultiTerminal).
  3. Run MultiTerminal.exe to launch the application.

No installer is required. MultiTerminal is a portable application — just extract and run. Configuration is stored in %APPDATA%\MultiTerminal.

First Launch

When you first launch MultiTerminal, you will see the main window with several dockable panels:

1. Terminal Panel

The central area hosts terminal tabs. Each tab is a full terminal emulator (ConPTY-based) where you can run Claude Code sessions or any shell command. Click the + button or use the Launcher panel to create a new terminal.

2. Kanban Board

The Tasks panel shows your kanban board with columns for To Do, In Progress, Done, and more. Tasks are persisted in SQLite and shared across all terminals.

3. Chat Panel

Send messages between terminals. When multiple Claude Code agents are running, they can communicate through this panel.

4. Other Panels

Use the toolbar buttons to toggle additional panels:

  • Activity — Real-time feed of task changes, messages, and agent events
  • Inbox — Personal notifications for task assignments and mentions
  • Profiles — Team member avatars, specialties, and availability
  • Projects — Project registry with paths, agents, and MCP servers
  • Launcher — Quick-launch terminals with pre-configured settings
  • Agents — Monitor and manage spawned team agents

Key Concepts

Terminals

Each terminal tab hosts a ConPTY-based shell rendered via WebView2. Terminals can register with the MultiTerminal system to gain an identity (name) for messaging and task management. A registered terminal appears in the team roster and can send/receive messages.

Kanban Board

The built-in task tracker uses a kanban workflow: todo → in_progress → review → done. Tasks support checklists (with per-item status tracking), plans, continuation notes, and helper assignments. All data is stored in SQLite (tasks.db).

Messaging

The MessageBroker routes messages between terminals. Messages can be direct (one-to-one) or broadcast (one-to-all). Claude Code agents use MCP tools to send and receive messages, enabling multi-agent collaboration.

Projects

Projects are registered in SQLite with associated paths, agents, MCP servers, prompts, and skills. The project context API (GET /api/projects/{id}/context) provides a complete snapshot for agent consumption.

Agents

MultiTerminal can spawn Claude Code agent teammates using the native agent teams feature. Agents are visible in the Agent panel, where you can monitor their transcript output in real time. Agents coordinate through the messaging system and work on kanban tasks.

Knowledge Base

The institutional memory system stores code digests, session history, and knowledge entries in SQLite. Agents can query the knowledge base for context about the codebase, past decisions, and established patterns.
